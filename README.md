1. How can I ensure that my code, program, or software is functional and secure?
Answer: I can make sure that the code is functional and secure by looking at what the code coverage for each method is.
If it is passing, I can almost for certain assume that it is at the very lease functional.
To ensure that it is secure, I would have to dive into each part of the function to make sure that it is not leaking anything that could be harmful to the consumer and/or developer.

5. How do I approach designing software?
Answer: The way that I approach designing software is first asking what the user needs are.
This would first give me what the ask is for and why we, as developers, are making changes to the code in the first place.
I would then ask myself what kind of tests I would need to make this logic functional and accurate. I personally believe that code can be created when test driven development is adopted, so I would first write the tests.
I would then write the logic and tweak it until it works as expected.
